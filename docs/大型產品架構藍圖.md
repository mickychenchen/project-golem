# 大型產品架構藍圖（Project Golem）

## 目標

將現有單體專案演進為可長期維護的產品級架構，兼顧：

- 模組邊界清楚
- 團隊可平行開發
- 啟動與部署一致
- 低風險漸進重構

## 當前分層

```text
project-golem/
├── apps/
│   ├── runtime/
│   │   └── index.js
│   └── dashboard/
│       └── plugin.js
├── src/
├── web-dashboard/
├── packages/
├── infra/
├── index.js
└── dashboard.js
```

### 邊界定義

- `apps/`：組裝層與啟動流程，不承載深業務邏輯。
- `src/`：核心領域邏輯（brain/memory/skills/managers）。
- `web-dashboard/`：Web UI 與 API adapter。
- `packages/`：可獨立測試與版本化的共用模組。
- `infra/`：部署、運維、觀測與環境治理。

## 依賴方向規範（必須遵守）

允許：

- `apps/* -> src/*`
- `apps/* -> packages/*`
- `web-dashboard/* -> src/*`（逐步改成 `packages/*`）

禁止：

- `src/* -> apps/*`
- `packages/* -> apps/*`
- `packages/*` 彼此循環依賴

## 命名與責任建議

- `apps/runtime`: orchestration only
- `src/core`: domain orchestration
- `src/managers`: policy / coordination
- `src/services`: integration helpers
- `src/utils`: pure/shared utilities

## 驗收準則

- 不改既有啟動命令即可正常啟動
- 核心測試持續通過
- 文件同步更新
- 新增功能優先放到正確層級，而非回填到入口大檔
