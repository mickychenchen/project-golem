# 開發者實作指南

本文提供 Project Golem 目前版本的實作入口與常見開發流程。

## 1. 啟動入口與主流程

- 相容入口：`/index.js`
- 實際入口：`/apps/runtime/index.js`
- 主要資料流：
  1. 平台事件進入 `UniversalContext`
  2. 送入 `ConversationManager`（防抖與排隊）
  3. 交給 `GolemBrain` 操作 Gemini Web
  4. 回覆經 `NeuroShunter` 分流成 Reply / Memory / Action
  5. Action 由 `TaskController` 與 `ActionQueue` 執行

## 2. 主要模組位置

- App 層：`/apps/runtime`、`/apps/dashboard`
- 核心：`/src/core`
- 管理器：`/src/managers`
- 技能：`/src/skills`
- 記憶：`/packages/memory`（source-of-truth，含 `embeddings/`）
- 共用套件：`/packages`（`security`、`protocol`、`memory` 已完成遷移）
- Dashboard 後端：`/web-dashboard/server.js`
- Dashboard 前端：`/web-dashboard/src`

## 3. 新增技能（Skill）流程

1. 在 `src/skills/core` 建立 `<skill-name>.js`。  
2. 模組需匯出 `name` 與 `run()`。  
3. 若需讓協議層讀到說明，補上 `src/skills/lib/<skill-name>.md`。  
4. 透過 `SkillManager.refresh()` 或重啟載入。  

## 4. 協議格式（GOLEM_PROTOCOL）

AI 回應由 `ResponseParser` 解析，主要區塊如下：

- `[GOLEM_MEMORY]`
- `[GOLEM_REPLY]`
- `[GOLEM_ACTION]`（JSON code block）

相關位置：

- `packages/protocol/ProtocolFormatter.js`（source-of-truth）
- `src/utils/ResponseParser.js`
- `packages/protocol/NeuroShunter.js`（source-of-truth）

## 5. 本地開發與測試

- 安裝：`npm install`
- 開發模式：`npm run dev`
- 單元測試：`npm test`
- 安全測試：`npm run test:security`
- 架構邊界檢查：`npm run arch:check`

建議 Node 版本依照 `.nvmrc`（20）執行，以降低版本差異問題。
